# Mining Face Generation Patterns / 채굴면 생성 패턴

## 1. Purpose

This document defines how a 3x3 Mining Face is generated by depth layer.

The goal is to prevent random or inconsistent block generation during implementation.

## 2. Core Rule

A Mining Face is always a 3x3 block wall.

```ts
type MiningFace = {
  depth: number;
  layerId: LayerId;
  visualPreset: FaceVisualPreset;
  cells: MiningFaceCell[];
};
```

Each face has exactly 9 cells.

```ts
cells.length = 9
```

Each cell has:

```ts
type MiningFaceCell = {
  index: 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8;
  blockType: BlockType;
  resourceId?: ResourceId;
  requiredHits: number;
  currentHits: number;
  isBroken: boolean;
};
```

## 3. Layer Depth Ranges

| Layer | Depth |
|---|---:|
| Dirt Layer | 0m ~ 49m |
| Stone Layer | 50m ~ 199m |
| Coal Vein | 200m ~ 449m |
| Iron Depth | 450m ~ 899m |
| Crystal Cave | 900m ~ 1499m |
| Diamond Depth | 1500m+ |
| Diamond Core Face | chapterClearDepthM |

Diamond Core Face at exactly chapterClearDepthM uses the fixed special layout.

## 4. Base Block by Layer

HARD_STONE_BLOCK / Hard Rock is removed from normal MVP generation.
It was a durable blocker that took more hits while still giving STONE, so it felt like an accidental resource tier between STONE and COAL.

| Layer | Base Blocks |
|---|---|
| Dirt Layer | DIRT_BLOCK, STONE_BLOCK |
| Stone Layer | STONE_BLOCK |
| Coal Vein | STONE_BLOCK |
| Iron Depth | STONE_BLOCK |
| Crystal Cave | STONE_BLOCK |
| Diamond Depth | STONE_BLOCK |
| Diamond Core Face | fixed layout |

## 5. Base Block Weight by Layer

| Layer | DIRT | STONE |
|---|---:|---:|
| Dirt Layer | 80 | 20 |
| Stone Layer | 0 | 100 |
| Coal Vein | 0 | 100 |
| Iron Depth | 0 | 100 |
| Crystal Cave | 0 | 100 |
| Diamond Depth | 0 | 100 |

Rules:

```text
STONE is the base wall material throughout the MVP.
COAL is the next resource tier after STONE.
There is no extra hard-rock material tier between STONE and COAL.
Do not generate HARD_STONE_BLOCK in normal Mining Faces.
```

## 6. Ore Pool by Layer

Ore unlocks are additive.
A deeper layer unlocks new ore types; it must not remove previously unlocked lower-tier ores.

| Layer | Ore Pool |
|---|---|
| Dirt Layer | COAL |
| Stone Layer | COAL, COPPER |
| Coal Vein | COAL, COPPER, IRON |
| Iron Depth | COAL, COPPER, IRON, GOLD |
| Crystal Cave | COAL, COPPER, IRON, GOLD, CRYSTAL |
| Diamond Depth | COAL, COPPER, IRON, GOLD, CRYSTAL, DIAMOND |

Rules:

```text
New layers add new ores.
New layers do not delete lower-tier ores.
Lower-tier ores should become less common as depth increases, but they must not drop to 0 once unlocked.
This is important because Mining Core nodes continue to require mixed lower-tier and higher-tier resources.
```

## 7. Ore Weight by Layer

Ore weights define the resource type chosen after an ore slot exists.  
They do not add new tuning variables.

Early coal must be more available than previous drafts because C1 Mining Core costs use COAL early.
Deeper layers must still contain lower-tier ores because later Mining Core costs use mixed materials.

| Layer | COAL | COPPER | IRON | GOLD | CRYSTAL | DIAMOND |
|---|---:|---:|---:|---:|---:|---:|
| Dirt Layer | 100 | 0 | 0 | 0 | 0 | 0 |
| Stone Layer | 75 | 25 | 0 | 0 | 0 | 0 |
| Coal Vein | 55 | 30 | 15 | 0 | 0 | 0 |
| Iron Depth | 20 | 25 | 40 | 15 | 0 | 0 |
| Crystal Cave | 12 | 18 | 30 | 25 | 15 | 0 |
| Diamond Depth | 8 | 12 | 20 | 25 | 25 | 10 |

Dirt Layer coal appears only when an ore slot/preset creates ore, but when it appears it is COAL.
This supports the first paid Mining Core unlocks without adding new catch-up variables.

Do not interpret deeper layers as replacing older ores.
For example, Iron Depth still contains COAL and COPPER, Crystal Cave still contains COAL/COPPER/IRON, and Diamond Depth still contains every unlocked ore type.

## 8. Ore Slot Count by Face Type

Face visual presets define how many ore cells appear.

```ts
type FaceVisualPreset =
  | 'PLAIN'
  | 'ORE_SPOT'
  | 'VEIN_HORIZONTAL'
  | 'VEIN_VERTICAL'
  | 'VEIN_L'
  | 'CLUSTER'
  | 'RICH_CROSS'
  | 'RARE_CENTER'
  | 'DIAMOND_CORE';
```

| Preset | Ore Cell Count | Use |
|---|---:|---|
| PLAIN | 0~1 | common |
| ORE_SPOT | 1~2 | common |
| VEIN_HORIZONTAL | 2~3 | vein |
| VEIN_VERTICAL | 2~3 | vein |
| VEIN_L | 3~4 | vein |
| CLUSTER | 3~4 | ore cluster |
| RICH_CROSS | 4~5 | rich face |
| RARE_CENTER | 1 rare center + 1~2 support | rare face |
| DIAMOND_CORE | fixed | chapterClearDepthM only |

Visual rendering rule:
Presets define mineral distribution inside the wall. They do not define icons or marks placed on top of cells.

- ORE_SPOT means a small local embedded mineral patch, not a centered dot.
- VEIN_HORIZONTAL, VEIN_VERTICAL, and VEIN_L mean irregular buried mineral seams continuing through neighboring cells, not smooth lines drawn above the wall.
- CLUSTER means several naturally embedded mineral fragments or exposed surfaces.
- RICH_CROSS means richer mineral distribution blended into cracks and chipped surfaces.
- RARE_CENTER means the center cell has a rare embedded material presence, not a centered symbol.
- DIAMOND_CORE means a fractured embedded core face, not a UI emblem.

Ore visuals should be irregular, partially occluded by rock, broken by cracks, and blended into the mine wall texture.

## 9. Preset Weights by Layer

Preset weights control how often ore-bearing layouts appear.  
Early layers should provide enough COAL opportunities for early Mining Core unlocks.
Deeper layers should have a higher chance of ore-bearing layouts because later Mining Core nodes require a wider mixed material pool.

| Layer | PLAIN | ORE_SPOT | VEIN | CLUSTER | RICH_CROSS | RARE_CENTER |
|---|---:|---:|---:|---:|---:|---:|
| Dirt Layer | 50 | 40 | 10 | 0 | 0 | 0 |
| Stone Layer | 35 | 40 | 20 | 5 | 0 | 0 |
| Coal Vein | 22 | 30 | 32 | 12 | 4 | 0 |
| Iron Depth | 18 | 25 | 30 | 16 | 10 | 1 |
| Crystal Cave | 12 | 20 | 30 | 18 | 17 | 3 |
| Diamond Depth | 8 | 18 | 28 | 18 | 20 | 8 |

VEIN weight is split between VEIN_HORIZONTAL, VEIN_VERTICAL, and VEIN_L.

Rules:

```text
As depth increases, ore-bearing faces become more common.
This does not mean every face is rich, but the average ore cell count should trend upward by layer.
Deep layers need more ore opportunities because the resource pool is wider and Mining Core costs require mixed materials.
```

Do not add separate early-coal guarantee variables for MVP.  
Use these existing layer ore weights and preset weights first.

## 10. Pattern Transform Rules

Patterns may be transformed to create variety.

Allowed transforms:

```ts
type PatternTransform =
  | 'NONE'
  | 'ROTATE_90'
  | 'ROTATE_180'
  | 'ROTATE_270'
  | 'MIRROR_HORIZONTAL'
  | 'MIRROR_VERTICAL';
```

Rules:

- transform must stay inside 3x3 grid
- transformed cell indices must remain unique
- Diamond Core layout does not transform
- generated face must remain deterministic from faceSeed

## 11. Deterministic Generation

Face generation should be deterministic.

Recommended seed:

```ts
faceSeed = hash(saveId, depth, faceIndex)
```

Given the same save state and depth, generation should be stable.

Do not use unseeded random values inside render components.

## 12. Diamond Core Face Layout

At chapterClearDepthM:

```text
0 STONE_BLOCK          1 DIAMOND_ORE_BLOCK   2 STONE_BLOCK
3 DIAMOND_ORE_BLOCK   4 DIAMOND_CORE_BLOCK  5 DIAMOND_ORE_BLOCK
6 STONE_BLOCK          7 DIAMOND_ORE_BLOCK   8 STONE_BLOCK
```

Clear condition:

```text
all 9 cells are EMPTY
```

Reward:

```text
DIAMOND +10
Chapter Clear
```


## 13. Natural Pattern Readability Rule

Ore distribution must not be interpreted as a simple mark stamped onto a cell.

Rules:

```text
ore patterns are irregular and embedded in rock
ore patches should not appear as the same round dot or stamp in a corner
ore veins should follow cracks, seams, exposed surfaces, or cell-to-cell rock structure
ore-bearing cells must remain visually distinguishable from plain rock
resource type must be identifiable from the wall itself, not only from toast/resource gain text
neighboring ore cells may visually connect through shared seams or broken surfaces
```

Pattern variety rules:

```text
Do not interpret this as banning every central shadow or soft cloudy highlight.
Those can be used as natural rock lighting or exposed stone texture.
What is banned is using the same centered mark as the ore/resource indicator on every block.
The generated 3x3 face needs varied but coherent rock patterns.
Vary crack direction, mineral seam shape, chip placement, edge damage, and exposure size by cell.
When a vein preset connects cells, the connection should look continuous, not copied.
```

Bad examples:

```text
same centered circle repeated as the ore marker
small round spot in upper-left repeated as the ore marker
smooth bright line floating above the wall
flat icon pasted over the block
generic blur/noise that hides the resource type
all cells using the same stamped pattern
```

Good examples:

```text
embedded mineral streaks
irregular fragments exposed by cracks
rough seams that continue across neighboring cells
chipped surfaces revealing mineral layers
small glints only on mineral surfaces
```

## 14. Validation

Manual validation:

- every face has exactly 9 cells
- layer depth selects the correct layer
- base block weights match the layer
- ore pool never uses unavailable future resources
- deeper layers keep lower-tier ores available after they have been unlocked
- average ore-bearing opportunity trends upward by depth layer
- Chapter Core Face appears at chapterClearDepthM
- Diamond Core Face is fixed and not transformed
- HARD_STONE_BLOCK is not generated in normal or core faces
- same seed generates the same face
- visual preset matches MiningFaceVeinOverlay
